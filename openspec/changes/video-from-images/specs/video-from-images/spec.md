# video-from-images

Users can create an MP4 slideshow video from selected images with configurable slide duration and optional background music. The video is generated by FFmpeg and saved as a cataloged asset.

---

## ADDED Requirements

### Requirement: A slideshow video can be created from selected images

`POST /api/assets/create-video` SHALL accept an ordered list of asset IDs, a slide duration, an optional music asset ID, and an output folder path. The endpoint SHALL invoke FFmpeg to produce an MP4 file and stream progress via SSE.

#### Scenario: Video created from images without music

- **GIVEN** 10 images selected in order with `slideDuration = 3` seconds and no music
- **WHEN** `POST /api/assets/create-video` is called
- **THEN** the endpoint returns an SSE stream; the generated video is 30 seconds long; SSE events report progress from PREPARING through ENCODING to DONE

#### Scenario: Video created with background music

- **GIVEN** 10 images and a music asset ID pointing to an MP3 file
- **WHEN** `POST /api/assets/create-video` is called with `musicAssetId`
- **THEN** the output MP4 contains the image slideshow with the MP3 audio; the video duration matches the shorter of the slideshow or music length (`-shortest` FFmpeg flag)

### Requirement: Output video is auto-cataloged as a new asset

When video generation completes successfully, the output MP4 SHALL be cataloged as a new asset in the specified output folder and returned in the DONE SSE event.

#### Scenario: Generated video appears in the gallery

- **GIVEN** video generation has completed
- **WHEN** the user navigates to the output folder in the gallery
- **THEN** the generated MP4 is visible as a video asset with a thumbnail

### Requirement: Individual image staging failure does not silently produce a corrupt video

If an image in the selected list cannot be read or staged, the operation SHALL fail with a descriptive error event on the SSE stream.

#### Scenario: Missing source image causes operation to fail

- **GIVEN** one of the selected asset IDs refers to a file that has been deleted from disk
- **WHEN** `POST /api/assets/create-video` is called
- **THEN** the SSE stream sends an `error` event with the message identifying the missing file; no video file is produced
