# audio-asset-support

Audio files (.mp3, .flac, .wav, .aac, .ogg) are cataloged as first-class assets. Audio metadata (title, artist, album, duration, bitrate) is extracted and displayed. The thumbnail is the embedded album art or a waveform image.

---

## Requirement: Audio files are cataloged with metadata

The catalog service SHALL process audio files with extensions `.mp3`, `.flac`, `.wav`, `.aac`, `.ogg` and store metadata in `asset_audio`.

### Scenario: MP3 with ID3 tags is cataloged

- **GIVEN** an MP3 file with ID3 tags: title "Song Title", artist "Artist Name", album "Album Name", duration 240s
- **WHEN** the catalog runs
- **THEN** an `assets` row is created with `fileType = AUDIO`; an `asset_audio` row is created with the extracted metadata

### Scenario: Audio file without tags is cataloged with null metadata

- **GIVEN** a WAV file with no embedded metadata
- **WHEN** the catalog runs
- **THEN** an `assets` row is created; the `asset_audio` row has `null` for title, artist, and album; `duration_seconds` is populated from the WAV header if readable

---

## Requirement: Embedded album art is used as the thumbnail

When an audio file contains embedded album art, it SHALL be used as the 200×150 px thumbnail.

### Scenario: Album art extracted as thumbnail

- **GIVEN** an MP3 with an embedded JPEG album art image
- **WHEN** the catalog processes the file
- **THEN** the thumbnail stored in `thumbnailUrl` is the album art scaled to 200×150 px

### Scenario: Waveform fallback when no album art

- **GIVEN** an MP3 with no embedded album art and FFmpeg is available
- **WHEN** the catalog processes the file
- **THEN** the thumbnail is a 200×150 waveform PNG generated by FFmpeg

---

## Requirement: Audio assets are displayed with a player in the gallery viewer

When the user opens an audio asset in the viewer, the frontend SHALL render `<audio controls>` with the audio stream URL and display title, artist, album, duration, and bitrate.

### Scenario: Audio viewer renders playback controls

- **GIVEN** a cataloged MP3 asset is opened in the viewer
- **WHEN** the viewer renders
- **THEN** an `<audio controls>` element is shown with the streaming URL, and the title, artist, album, duration, and bitrate are displayed beneath
